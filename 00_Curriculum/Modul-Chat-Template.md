# Modul-Chat Template

**Verwende diesen Prompt, wenn du einen neuen Chat für ein Modul startest.**

---

## Template-Prompt für Modul X

```
Kontext: Ich arbeite an meinem Claude Code Masterprogramm für Agentic AI.

Ziel: Top 1% Experte werden + Schulungsmaterialien erstellen
Profil: 20 Jahre IT, Softwareentwickler & Cloud Architekt
Setup: Claude Code, VS Code, Claude Pro
Lernzeit: 4-6h/Tag (Vollzeit)

Aktueller Stand:
- Abgeschlossene Module: [Liste der Module, z.B. "1, 2, 3"]
- Aktuelles Modul: Modul X - [Titel]
- Modul-Nummer: X von 12

Lernziele dieses Moduls (aus Curriculum):
[Kopiere die Lernziele aus dem Gesamtcurriculum]

Erwartete Dauer: [X] Tage
Schwierigkeit: [X]/5

Anweisungen:
1. Starte mit einer kurzen Wiederholung des vorherigen Moduls (1-2 Min Lesezeit)
2. Führe mich durch die Theorie (kompakt, präzise, für meinen Level)
3. Gib mir praktische Beispiele
4. Stelle mir die Challenge
5. Stelle mir Praxisaufgaben
6. Am Ende: Modul-Prüfung

Lernstil:
- Präzise und direkt (kein Füllmaterial)
- Praxis-fokussiert (70% Praxis, 30% Theorie)
- Architektur-Perspektive integrieren
- Schritt-für-Schritt, aber zügig (kein Anfänger-Tempo)

Dokumentation:
Erstelle am Ende des Moduls:
- Modul_X_Theorie.docx (Schulungsunterlagen)
- Modul_X_Übungen.md (Alle praktischen Aufgaben mit Lösungen)
- Modul_X_Challenge.md (Challenge-Beschreibung + Musterlösung)
- Modul_X_Prüfung.docx (Prüfungsfragen + Lösungen)
- Modul_X_Abschluss.md (Zusammenfassung + Überleitung zu Modul X+1)

Bitte starte jetzt mit Modul X.
```

---

## Anpassungen pro Modul

### Für Modul 1 (erster Start):
- Keine "Wiederholung des vorherigen Moduls" nötig
- Beginne direkt mit einer Einleitung ins Thema

### Für Modul 2-12:
- Füge immer die Liste der abgeschlossenen Module hinzu
- Kopiere die spezifischen Lernziele aus dem Curriculum
- Erwähne relevante Erkenntnisse aus vorherigen Modulen, falls relevant

### Bei Schwierigkeiten während eines Moduls:
Füge hinzu:
```
Zusätzlicher Kontext:
Ich hatte Schwierigkeiten mit [Thema]. Bitte passe die Erklärung entsprechend an.
```

### Bei zu langsamem Tempo:
Füge hinzu:
```
Tempo-Anpassung:
Das letzte Modul war zu einfach. Erhöhe die Komplexität für dieses Modul.
```

### Bei zu schnellem Tempo:
Füge hinzu:
```
Tempo-Anpassung:
Das letzte Modul war sehr anspruchsvoll. Lass uns etwas langsamer vorgehen.
```

---

## Workflow nach Modul-Abschluss

1. **Im Modul-Chat:**
   - Modul vollständig durcharbeiten
   - Alle Aufgaben lösen
   - Prüfung ablegen
   - Dateien werden erstellt

2. **Zurück im Master-Chat:**
   - Melden: "Modul X abgeschlossen"
   - Fortschritts-Tracker wird aktualisiert
   - Optionales Feedback geben
   - Übergang zu Modul X+1 vorbereiten

3. **Neuer Chat für Modul X+1:**
   - Template-Prompt kopieren
   - Anpassen für Modul X+1
   - Starten

## Tipps für effektive Modul-Durchführung

### Vor dem Modul:
✅ Lies das Curriculum für dieses Modul nochmal durch
✅ Überprüfe, ob alle vorherigen Module abgeschlossen sind
✅ Plane Zeit ein (3-6 Tage je nach Modul)

### Während des Moduls:
✅ Mache Notizen zu Schwierigkeiten
✅ Frage nach, wenn etwas unklar ist
✅ Ergänze Beispiele aus deiner eigenen Erfahrung
✅ Experimentiere über die Aufgaben hinaus

### Nach dem Modul:
✅ Reflektiere: Was war einfach? Was war schwer?
✅ Update den Fortschritts-Tracker
✅ Gehe zurück zum Master-Chat für Übergang
✅ Mache eine kurze Pause vor dem nächsten Modul (wenn nötig)

---

## Häufige Anpassungen

**"Ich brauche mehr Beispiele"**
→ Füge hinzu: "Bitte gib mir 2-3 zusätzliche praktische Beispiele für jedes Konzept."

**"Ich möchte tiefer eintauchen"**
→ Füge hinzu: "Bitte gehe tiefer auf [Thema] ein. Ich möchte die Details verstehen."

**"Das ist zu akademisch"**
→ Füge hinzu: "Fokus auf praktische Anwendung, weniger Theorie."

**"Ich möchte das im Kontext von [X] verstehen"**
→ Füge hinzu: "Bitte erkläre die Konzepte im Kontext von [Cloud Architecture / E-Commerce / FinTech / etc.]"

---

**Du kannst dieses Template beliebig anpassen. Der Kern ist: Claude braucht Kontext, Ziel und klare Anweisungen.**
